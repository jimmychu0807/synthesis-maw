## Rules
1. Ship something that works — demos, prototypes, deployed contracts. Ideas alone don't win.
2. Agent must be a real participant — not a wrapper. Show meaningful contribution.
3. On-chain artifacts count — contracts, ERC-8004 registrations, attestations. More = stronger.
4. Open source required — all code public by deadline.
5. Document your process — use `conversationLog` field. Brainstorms, pivots, breakthroughs.

---

## Themes

### 1. Agents that keep secrets (PRIMARY — matches our project)
Private payment rails, ZK authorization, encrypted agent-to-service communication, human-controlled disclosure.

### 2. Agents that pay (SECONDARY — matches our project)
Scoped spending permissions, on-chain settlement, conditional payments/escrow, auditable history.

### 3. Agents that trust
On-chain attestations, portable credentials, open discovery, verifiable service quality.

### 4. Agents that cooperate
Smart contract commitments, negotiation boundaries, dispute resolution, composable coordination primitives.
