## Judging Principles (from themes.md)
1. "Start from a real problem" — builders who've felt the pain firsthand
2. "Build for the human, not the agent" — human stays in control
3. "Use what already exists" — connect existing tools to agent use cases in ways no one has tried
4. "Solve a problem, not a checklist" — integrating five tools that don't add up isn't a project
5. "Don't over-scope" — working demo of one well-scoped idea beats ambitious architecture diagram
6. Ship something that works — demos, prototypes, deployed contracts
7. Agent must be real participant — not a wrapper, show meaningful contribution
8. Everything on-chain counts — more on-chain artifacts = stronger
9. Open source required
10. Document your process — conversationLog field
